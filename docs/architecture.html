Architecture架构说明

Design Principle 设计理念

HaeronClaw separates the authoring surface (what you write) from the hosting runtime (how it runs in the cloud). The src/ folder is the agent itself — instructions, skills, MCP config, tools, and knowledge. The infra/assets/ folder is the Azure-specific runtime that wraps that agent for cloud execution. HaeronClaw 将创作面（你写的内容）和托管运行时（云端如何运行）分离。src/ 目录是 agent 本身 — 指令、skills、MCP 配置、工具和知识。infra/assets/ 目录是将 agent 封装为云端服务的 Azure 运行时。

Three Surfaces 三个面

✍️

Authoring Surface创作面

AGENTS.md, skills, MCP servers, Python tools, work memory. You write these in VS Code and test locally with Copilot Chat. AGENTS.md、skills、MCP servers、Python 工具、work memory。在 VS Code 中编写，本地用 Copilot Chat 测试。

⚡

Delivery Surface交付面

HTTP API, SSE streaming, browser Chat UI, Teams bot, MCP endpoint. The same agent is accessible through multiple channels. HTTP API、SSE 流式、浏览器聊天 UI、Teams bot、MCP 端点。同一个 agent 可通过多个渠道访问。

☁️

Deployment Surface部署面

Single deploy.ps1 entry point provisions ACR, ACA, Storage, and optionally a Foundry account. Infrastructure is defined in Bicep under infra/. 统一的 deploy.ps1 入口预置 ACR、ACA、Storage，可选部署 Foundry 账户。基础设施通过 infra/ 下的 Bicep 定义。

Bundled Dependency Patches 打包依赖补丁

The repository patches selected m365-cli files used by the Azure runtime. Source-controlled overrides live under infra/assets/vendor/m365-cli/. The post-install script apply-m365-cli-patches.mjs copies them into node_modules/ after npm install. 仓库对 Azure 运行时使用的部分 m365-cli 文件做定制。版本控制中的覆盖文件位于 infra/assets/vendor/m365-cli/。apply-m365-cli-patches.mjs 在 npm install 后自动复制到 node_modules/。

Known Limitations 已知限制

  • Python tools in src/tools/ do not work locally — they are fully functional only after deploying to ACA. src/tools/ 下的 Python 工具在本地不可用 — 仅部署到 ACA 后才能正常工作。
  • Windows is not supported for packaging hooks — the shell scripts require macOS, Linux, or WSL. Windows 不受支持用于打包 hook — shell 脚本需要 macOS、Linux 或 WSL。

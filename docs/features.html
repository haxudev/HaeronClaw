Cloud-Hosted Runtime云托管运行时

HaeronClaw deploys to Azure Functions on Azure Container Apps. After deployment, the agent is available as: HaeronClaw 部署到 Azure Container Apps 上的 Azure Functions。部署后，agent 以以下方式提供服务：

  • HTTP API — POST /agent/chat (JSON) and POST /agent/chatstream (SSE)HTTP API — POST /agent/chat（JSON）和 POST /agent/chatstream（SSE）
  • Chat UI — Built-in single-page chat interface at the root URL聊天 UI — 根 URL 提供的内置单页聊天界面
  • MCP endpoint — /runtime/webhooks/mcp for remote MCP clientsMCP 端点 — /runtime/webhooks/mcp 供远程 MCP 客户端连接

The same source code runs locally in VS Code Copilot Chat and in the cloud — no rewrites needed. 同一份代码在本地 VS Code Copilot Chat 和云端运行，无需重写。

Remote MCP Endpoint远程 MCP 端点

The deployed app exposes an MCP server at /runtime/webhooks/mcp. Configure it in VS Code: 部署后应用在 /runtime/webhooks/mcp 暴露 MCP server。在 VS Code 中配置：

json 
{
  "inputs": [
    {
      "type": "promptString",
      "id": "functions-mcp-extension-system-key",
      "description": "Azure Functions MCP Extension System Key",
      "password": true
    },
    {
      "type": "promptString",
      "id": "functionapp-host",
      "description": "Container app host"
    }
  ],
  "servers": {
    "remote-mcp-function": {
      "type": "http",
      "url": "https://${input:functionapp-host}/runtime/webhooks/mcp",
      "headers": {
        "x-functions-key": "${input:functions-mcp-extension-system-key}"
      }
    }
  }
}

Teams IntegrationTeams 集成

HaeronClaw integrates deeply with Microsoft Teams: HaeronClaw 深度集成 Microsoft Teams：

  • Async execution — Long-running jobs are processed in the background异步执行 — 长耗时任务在后台处理
  • Proactive replies — Results delivered back to the conversation automatically主动回复 — 结果自动回送到对话中
  • Typing indicators — Visual feedback while the agent is workingTyping indicator — agent 工作时显示视觉反馈
  • Heartbeat progress — Periodic status updates for long tasks心跳进度 — 长任务的周期性状态更新
  • Voice transcription — Audio attachments transcribed via Azure Speech语音转录 — 音频附件通过 Azure Speech 转录

Artifact Delivery制品交付

Generated files (PPTX, DOCX, XLSX, PDF, etc.) are automatically uploaded to Azure Blob Storage and exposed through application-owned download links. No SAS token handling required by the user. 生成的文件（PPTX、DOCX、XLSX、PDF 等）会自动上传到 Azure Blob Storage，并通过应用自有下载链接对外提供，用户无需处理 SAS token。

Binary email attachments are also staged to Blob Storage and rewritten as clean download links when direct attachment delivery is not appropriate. 二进制邮件附件在不适合直接投递时，也会暂存到 Blob Storage，并重写为整洁的下载链接。

Session Persistence会话持久化

When running in Azure, agent sessions are automatically persisted to an Azure Files share mounted into the container app. Conversation state survives restarts and is shared across instances. 在 Azure 中运行时，agent 会话自动持久化到挂载到容器应用的 Azure Files 共享。会话状态跨重启保留，并在多实例间共享。

Locally, sessions are stored in ~/.copilot/session-state/. 本地运行时，会话保存在 ~/.copilot/session-state/。

Knowledge-Backed Responses知识增强回答

Internal work_memory/ content and Microsoft Learn sources can be combined for source-grounded answers. The agent retrieves from indexed content and provides citations. 内部 work_memory/ 内容与 Microsoft Learn 文档可以组合，输出带来源依据的回答。

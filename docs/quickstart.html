Quickstart快速开始

Running Locally in VS Code在 VS Code 中本地运行

The fastest way to interact with HaeronClaw is to run it locally in VS Code as a standard Copilot project. 最快的上手方式是在 VS Code 中作为标准 Copilot 项目本地运行。

  1. Open the src folder in VS Code 在 VS Code 中打开 src 目录
  2. Enable the experimental setting: chat.useAgentSkills 启用实验性设置：chat.useAgentSkills
  3. Enable built-in tools in Copilot Chat 在 Copilot Chat 中启用内置工具
  4. Start chatting with your agent in Copilot Chat 在 Copilot Chat 中与 agent 对话

Note: AGENTS.md instructions, .github/skills/ skills, and .vscode/mcp.json MCP servers are all automatically loaded. Python tools in src/tools/ require cloud deployment. 注意：AGENTS.md 指令、.github/skills/ skills 和 .vscode/mcp.json MCP servers 都会自动加载。src/tools/ 下的 Python 工具需要部署到云端才可使用。

Deploying to Azure Functions on ACA部署到 Azure Functions on ACA

Use the unified deployment entry point to deploy HaeronClaw to Azure Container Apps: 使用统一部署入口将 HaeronClaw 部署到 Azure Container Apps：

powershell 
./scripts/deploy.ps1 -Mode aca -ResourceGroup <rg-name> -Location eastus2 -Prefix fmaaca -Model github:gpt-5.4 -ImageTag v3

The main deployment inputs are:主要部署参数如下：

Parameter参数 Description说明
-ResourceGroupTarget Azure resource group目标 Azure 资源组
-LocationAzure region for deployment部署区域
-PrefixNaming prefix for resources资源命名前缀
-ModelRuntime model, e.g. github:gpt-5.4运行时模型，如 github:gpt-5.4
-ImageTagContainer image tag容器镜像 tag

Within minutes you'll have a deployed agent behind an HTTP API and a built-in chat UI. The same source code that runs locally in Copilot Chat now runs remotely on Azure. 几分钟后，你会得到一个通过 HTTP API 暴露、并附带内置聊天 UI 的已部署 agent。

Model Configuration模型配置

The project uses a single environment variable COPILOT_MODEL at runtime. Deployment scripts normalize the -Model input into this variable automatically. 项目在运行时使用唯一环境变量 COPILOT_MODEL。部署脚本会自动将 -Model 输入归一化为该变量。

Category类别 Prefix前缀 Infrastructure基础设施 Examples示例
GitHub Models github: None (uses GitHub API)无（使用 GitHub API） github:gpt-5.4, github:claude-sonnet-4.6
Foundry Models foundry: Deploys Foundry account in your subscription在你的订阅中部署 Foundry 账户 foundry:gpt-4.1-mini, foundry:o4-mini

To change the model after deployment, re-run the deploy script with a new -Model value. 部署后切换模型，只需用新的 -Model 值重新运行部署脚本。

Timer Triggers定时触发

Define scheduled agent runs directly in src/AGENTS.md frontmatter: 在 src/AGENTS.md 的 frontmatter 中定义定时运行的 agent：

yaml 
---
functions:
  - name: timerAgent
    trigger: timer
    schedule: "0 */2 * * * *"
    prompt: "What's the price of a Standard_D4s_v5 VM in East US?"
    logger: true
---

Note: Only trigger: timer is supported. Other trigger types are rejected at startup. 注意：目前只支持 trigger: timer，其他触发器类型会在启动时被拒绝。

Custom Python Tools自定义 Python 工具

Add custom tools by dropping Python files into src/tools/: 将 Python 文件放入 src/tools/ 即可扩展工具能力：

python 
from pydantic import BaseModel, Field

class CostEstimatorParams(BaseModel):
    unit_price: float = Field(description="Retail price per unit")
    unit_of_measure: str = Field(description="Unit of measure, e.g. '1 Hour'")
    quantity: float = Field(description="Monthly quantity")

async def cost_estimator(params: CostEstimatorParams) -> str:
    """Estimate monthly and annual costs from unit price and usage."""
    monthly_cost = params.unit_price * params.quantity
    annual_cost = monthly_cost * 12
    return f"Monthly: ${monthly_cost:.4f} | Annual: ${annual_cost:.4f}"

The runtime scans tools/*.py, loads module-level functions (excluding _ prefixed), uses the docstring as the tool description, and registers one function per file. 运行时会扫描 tools/*.py，加载模块级函数（排除 _ 开头的名称），使用 docstring 作为工具描述，每个文件注册一个函数。
